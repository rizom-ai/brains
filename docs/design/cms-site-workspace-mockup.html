03

Live is explicit, not alarming

Click “Update live site” above. The confirmation names the public URL, content scope, and previous successful result without pretending the builder is a deployment platform.

Production side-effect guard Server rejects an unconfirmed action
Action label

Update, not deploy

The running app serves production output. The interface does not claim DNS, release promotion, or provider deployment work occurred.

Content boundary

Published + public

Preview may include public drafts. Live retains the production builder’s published-only and public-visibility filters.

Failure behavior

Keep the last proof

A failed rebuild reports an operator-safe error while preserving the last successful time and route count.

Execution path

One rebuild manager

CMS, tools, and auto-rebuild all use the same debounce, deduplication, and site-build job path.

04

A destination on mobile; no scar when absent

Site joins the existing horizontal switcher. Removing site-builder removes only Site; Publishing remains the selected operational workspace.

Information architecture

Glance, then manage

Dashboard reports health and attention. CMS owns rebuild actions, details, and the editing-to-preview loop.

Visual language

Proof → live

Two environment plates express scope and consequence more clearly than a generic job table or build-history dashboard.

State boundary

Builder owns truth

Dashboard and CMS render one safe snapshot. Neither scans output directories or interprets generic job records itself.

Optional composition

One provider can leave

Publishing and Site register independently. Removing either leaves the other workspace and ordinary editor routes intact.
