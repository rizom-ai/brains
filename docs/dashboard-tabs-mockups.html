Tabs are derived, not configured: widgets declare a group; the dashboard renders Overview plus one tab per non-empty group. Group ids are chosen to match the planned bundle vocabulary (site, publishing, team) so the two converge when bundles ship — the ghost “team” tab appears the day a team-group widget registers.
Badges are operator work, not noise: the solid badge counts items that need a decision (reviews, approvals, discovered agents); the outline badge is plain volume. Overview digests answer “anything need me?” in one glance; each digest is the door to its tab.
Still server-rendered: same Preact SSR + tiny inline script as today — tabs are progressive enhancement over anchor targets, no SPA. The console strip (Dashboard / Chat / CMS · ⌘K · session chip) is shared chrome across all three surfaces.
